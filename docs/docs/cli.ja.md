# Kube KTS CLI

Kube KTS は `kube-kts` コマンドで実行できるコマンドラインツールです。

リポジトリを処理するには、リポジトリのパスを指定します。リポジトリのパスを
指定しない場合は、現在の作業ディレクトリが使用されます。

## グローバルオプション

- `--debug`: ログレベルを含むデバッグ情報を出力します。
- `--verbose`: ログレベルを含むすべての情報を出力します。
- `--show-log-level`: 出力にログレベルを表示します。
- `--exception`: エラー発生時に完全な例外スタックトレースを出力します。
- `--experimental`: 実験的機能を有効にします。
- `--unsafe`: アンセーフモードを有効にし、Kotlin スクリプト内で `import` 文や完全修飾クラス名を使用できるようにします。

## 実験的機能

実験的機能は安定しておらず、将来のバージョンで変更される可能性があります。使用するには、
`--experimental` フラグを有効にします。

- `--yaml-merge`: YAML ファイルのマージに使用するアルゴリズムを指定します。
    - `HELM`: 標準の Helm マージアルゴリズムを使用します（デフォルト）。
    - `INTERNAL`: 独自の内部マージアルゴリズムを使用します。
- `--yaml-array-merge`: `INTERNAL` アルゴリズム使用時の配列のマージ戦略を指定します。
    - `None`: ベース配列を変更しません。
    - `Replace`: ベース配列をオーバーレイ配列で置き換えます（デフォルト）。
    - `AddFirst`: オーバーレイ配列をベース配列の先頭に追加します。
    - `AddLast`: オーバーレイ配列をベース配列の末尾に追加します。

## Validate（検証）

`kube-kts validate` コマンドはリポジトリを検証します。リポジトリが有効で、
必要なすべてのファイルを含んでいるかを確認します。

### パラメータ

1. `REPOSITORY`: リポジトリのパス。指定しない場合は現在のディレクトリが使用されます。

### 例

```bash
# 現在のディレクトリを検証する
kube-kts validate .

# 特定のリポジトリを検証する
kube-kts validate /path/to/repository
```

## Compile（コンパイル）

`kube-kts compile` コマンドはリポジトリをコンパイルします。リポジトリから
オブジェクトインスタンスを生成し、それを次のステップでテンプレートのレンダリングに使用します。

### パラメータ

1. `REPOSITORY`: リポジトリのパス。指定しない場合は現在のディレクトリが使用されます。

### 例

```bash
# 現在のディレクトリをコンパイルする
kube-kts compile .

# 特定のリポジトリをコンパイルする
kube-kts compile /path/to/repository
```

## Render（レンダリング）

`kube-kts render` コマンドはリポジトリをレンダリングします。リポジトリから
Helm ファイルを生成し、Helm で使用できるようにします。

### パラメータ

1. `REPOSITORY`: リポジトリのパス。指定しない場合は現在のディレクトリが使用されます。
2. `TARGET`: Helm ファイルをレンダリングする出力先ディレクトリのパス。指定しない場合は一時ディレクトリが使用されます。

### 例

```bash
# 現在のディレクトリを出力先ディレクトリにレンダリングする
kube-kts render . /path/to/target

# 特定のリポジトリを出力先ディレクトリにレンダリングする
kube-kts render /path/to/repository /path/to/target
```

## Lint（リント）

`kube-kts lint` コマンドは Helm を使ってリポジトリをリントします。レンダリングされた
Helm 出力が有効かどうかを確認します。

### パラメータ

1. `REPOSITORY`: リポジトリのパス。指定しない場合は現在のディレクトリが使用されます。
2. `TARGET`: リント前に Helm ファイルをレンダリングする出力先ディレクトリのパス。指定しない場合は一時ディレクトリが使用されます。

### 例

```bash
# まず出力先ディレクトリにレンダリングしてから現在のディレクトリをリントする
kube-kts lint . /path/to/target

# まず出力先ディレクトリにレンダリングしてから特定のリポジトリをリントする
kube-kts lint /path/to/repository /path/to/target
```

## Template（テンプレート）

`kube-kts template` コマンドはリポジトリのレンダリング済みテンプレートを出力します。内部的に `helm template` を使用します。

### パラメータ

1. `REPOSITORY`: リポジトリのパス。指定しない場合は現在のディレクトリが使用されます。
2. `TARGET`: Helm ファイルをレンダリングする出力先ディレクトリのパス。指定しない場合は一時ディレクトリが使用されます。

### オプション

- `-n`、`--name`: **（必須）**テンプレート化するチャートの名前。

### 例

```bash
# 特定の名前で現在のディレクトリをテンプレート化する
kube-kts template . /path/to/target --name my-release

# 特定の名前で特定のリポジトリをテンプレート化する
kube-kts template /path/to/repository /path/to/target -n my-release
```
