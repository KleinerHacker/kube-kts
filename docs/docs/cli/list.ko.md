# list

```bash
kube-kts list [옵션]
```

`helm list`를 실행하여 릴리스를 나열합니다. `status`와 마찬가지로 `list`는 클러스터를 대상으로 동작하므로 **저장소도 렌더링 단계도 필요하지 않습니다** — 호출은 Helm으로 직접 전달됩니다.

## 동작 방식

1. 저장소를 스캔·컴파일·렌더링하지 않습니다. KTS 스크립트는 여기서 무관합니다.
2. 전달되는 모든 옵션을 덧붙여 `helm list`를 실행합니다.
3. Helm이 릴리스 목록을 출력합니다. 명령의 종료 코드는 Helm 결과를 반영합니다.

## list 옵션

| 옵션 | 마커 | 설명 |
|---|---|---|
| `-a`, `--all` | `---->` | 필터를 적용하지 않고 모든 릴리스를 표시합니다. |
| `-A`, `--all-namespaces` | `---->` | 모든 네임스페이스의 릴리스를 나열합니다. |
| `-d`, `--date` | `---->` | 릴리스 날짜로 정렬합니다. |
| `--deployed` | `---->` | 배포된 릴리스를 표시합니다. 다른 상태 필터가 없으면 자동으로 활성화됩니다. |
| `--failed` | `---->` | 실패한 릴리스를 표시합니다. |
| `--filter=REGEXP` | `---->` | Perl 호환 정규식. 일치하는 릴리스만 나열합니다. |
| `-m`, `--max=INT` | `---->` | 가져올 릴리스의 최대 개수. |
| `--no-headers` | `---->` | 기본 출력 형식에서 헤더를 출력하지 않습니다. |
| `--offset=INT` | `---->` | 목록의 다음 릴리스 인덱스. 시작 위치에서의 오프셋에 사용됩니다. |
| `-o`, `--output=FORMAT` | `---->` | 출력 형식: `table`(기본값), `json`, `yaml`. |
| `--pending` | `---->` | 대기 중인 릴리스를 표시합니다. |
| `-r`, `--reverse` | `---->` | 정렬 순서를 반대로 합니다. |
| `-l`, `--selector=SELECTOR` | `---->` | 필터링용 레이블 쿼리(`=`, `==`, `!=` 지원). |
| `-q`, `--short` | `---->` | 짧은(조용한) 목록 형식으로 출력합니다. |
| `--superseded` | `---->` | 대체된 릴리스를 표시합니다. |
| `--time-format=FORMAT` | `---->` | Go 시간 포매터로 시간을 형식화합니다. |
| `--uninstalled` | `---->` | 제거된 릴리스를 표시합니다(제거 시 `--keep-history`를 사용한 경우). |
| `--uninstalling` | `---->` | 현재 제거 중인 릴리스를 표시합니다. |

## Helm 전역 옵션

모든 [Helm 전역 옵션](status.md#helm-global-options)(`-n`/`--namespace`, `--kube-context` 등)이 Helm으로 전달됩니다.

## 전역 옵션

모든 [전역 옵션](index.md)(`--debug`, `--unsafe` 등)도 사용할 수 있습니다.

## 예시

```bash
# 현재 네임스페이스의 릴리스 나열
kube-kts list

# 모든 네임스페이스의 전체 릴리스를 JSON으로 나열
kube-kts list -A -o json

# 실패한 릴리스만 최신순으로 표시
kube-kts list --failed --date --reverse
```
