# push

```bash
kube-kts push <CHART> <REMOTE> [オプション]
```

`helm push` を実行して、パッケージ化されたチャートをリモート（OCI）レジストリにアップロードします。既存のチャートパッケージに対して動作するため、**リポジトリもレンダリング手順も不要**です——呼び出しは直接 Helm に転送されます。

## 動作の仕組み

1. リポジトリのスキャン・コンパイル・レンダリングは行われません。KTS スクリプトはここでは無関係です。
2. `helm push <CHART> <REMOTE>` が、転送されるすべてのオプションを付加して実行されます。
3. Helm がチャートをアップロードします。コマンドの終了コードは Helm の結果を反映します。

## パラメータ

| パラメータ | 必須 | 説明 |
|---|---|---|
| `CHART` | はい | パッケージ化されたチャートのパス（`.tgz`）。位置引数 `CHART` として Helm に転送されます。 |
| `REMOTE` | はい | リモートレジストリ参照。位置引数 `REMOTE` として Helm に転送されます。 |

## push オプション

| オプション | マーカー | 説明 |
|---|---|---|
| `--ca-file=FILE` | `---->` | レジストリの TLS 証明書を検証する CA バンドル。 |
| `--cert-file=FILE` | `---->` | クライアントを識別する SSL 証明書ファイル。 |
| `--key-file=FILE` | `---->` | クライアントを識別する SSL 鍵ファイル。 |
| `--insecure-skip-tls-verify` | `---->` | アップロードの TLS 証明書チェックをスキップします。 |
| `--plain-http` | `---->` | アップロードに安全でない HTTP を使用します。 |

## Helm グローバルオプション

すべての [Helm グローバルオプション](status.md#helm-global-options)が Helm に転送されます。

## グローバルオプション

すべての[グローバルオプション](index.md)も利用できます。

## 例

```bash
# パッケージ化されたチャートを OCI レジストリにプッシュ
kube-kts push my-app-1.0.0.tgz oci://registry.example.com/charts
```
