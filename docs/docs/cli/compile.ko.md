# compile

```bash
kube-kts compile <REPOSITORY> [--unsafe]
```

KTS 저장소를 객체 인스턴스로 컴파일합니다. [`validate`](validate.md) 보다 한 단계 더 나아가, 실제로
Kotlin 스크립트를 실행하여 렌더러가 나중에 YAML 로 변환하는 Chart 의 메모리 내 객체 모델(chart,
deployment, service 등)을 구성합니다. 아직 출력을 쓰지 않으면서 스크립트가 **컴파일·실행**되는지(타입
오류, 널 안전 위반, 스크립트 내부에서 던져지는 런타임 예외 포함)를 확인하기에 적합한 명령입니다.

컴파일은 스크립트를 실행하므로 스크립트 수준의 문제가 여기서 먼저 드러납니다. 잘못된 값 조회, 실패한
`require(...)`, 가져오기가 허용되지 않은 클래스를 참조하는 스크립트 등입니다.

!!! warning "`--unsafe` 는 임의 코드를 실행합니다"
    기본적으로 스크립트는 `import` 문과 정규화된 클래스 이름을 금지하는 제한된 샌드박스에서 실행됩니다.
    필요하면 `--unsafe` 를 추가하되, 스크립트가 임의 JVM 코드를 실행할 수 있으므로 신뢰하는 저장소에서만
    사용하세요.

## 매개변수

| 매개변수 | 필수 | 설명 |
|---|---|---|
| `REPOSITORY` | 예 | 컴파일할 저장소 경로. 생략하면 현재 작업 디렉터리를 사용합니다. 존재하지 않는 경로는 즉시 실패합니다. |

## 옵션

[전역 옵션](index.md) 외에, 컴파일이 스크립트를 실행하는 단계이므로 [`--unsafe`](index.md) 플래그가
특히 관련됩니다.

| 옵션 | 마커 | 설명 |
|---|---|---|
| `--unsafe` | `!!!` | 스크립트에서 `import` 문과 정규화된 클래스 이름을 허용하여 추가 JVM 클래스를 사용할 수 있게 합니다. 임의 코드 실행을 가능하게 하므로 신뢰하는 저장소에서만 사용하세요. |

## 예시

```bash
# 현재 디렉터리 컴파일
kube-kts compile .

# 스크립트에서 import 를 사용하는 저장소 컴파일
kube-kts compile /path/to/repository --unsafe

# 스크립트가 예외를 던지면 전체 스택 트레이스 출력
kube-kts compile ./helm --exception
```
