# lint

```bash
kube-kts lint <REPOSITORY> [TARGET] [オプション]
```

リポジトリを Helm Chart にレンダリングし、それに対して `helm lint` を実行します。lint はレンダリング
済み Chart の問題やベストプラクティス違反——必須フィールドの欠落、不正な `Chart.yaml`、有効な YAML を
生成しないテンプレート、推奨だが欠けているメタデータなど——を調べ、情報・警告・エラーとして報告します。

Kube KTS が先にレンダリングするため、`lint` は値が適用済みの、Helm が実際に受け取る*本物の* Chart を
検証します。[`compile`](compile.md) の自然な次のステップです。compile はスクリプトが動くことを、lint は
出来上がった Chart が健全であることを証明します。

## 仕組み

1. リポジトリをスキャン・コンパイルし、（一時または明示の）Chart ディレクトリにレンダリングします。
2. そのディレクトリで `helm lint .` を実行し、転送されるオプションをすべて付加します。
3. Kube KTS と Helm の出力をまとめて表示します。コマンドの終了コードは Helm の結果を反映します。

!!! tip "警告でビルドを失敗させる"
    既定では `helm lint` はエラー時のみ失敗します。`--strict` を追加すると警告でも失敗し、通常 CI で
    望ましい挙動になります。

## パラメータ

| パラメータ | 必須 | 説明 |
|---|---|---|
| `REPOSITORY` | はい | lint するリポジトリのパス。省略時はカレントディレクトリを使用します。 |
| `TARGET` | いいえ | lint 前に Chart をレンダリングするディレクトリ。省略時は一時ディレクトリを使用します。 |

## lint オプション

| オプション | マーカー | 説明 |
|---|---|---|
| `--quiet` | `---->` | 情報メッセージを抑制し、警告とエラーのみ出力します。簡潔な CI ログに便利です。 |
| `--strict` | `---->` | 警告を失敗として扱い、どの lint 警告でも 0 以外の終了コードにします。スタイル/ベストプラクティスの問題を通したくないパイプラインに推奨です。 |
| `--with-subcharts` | `---->` | 最上位 Chart に加え、依存するサブチャートも lint します。 |

## 値オプション

Chart の値を設定/上書きし、Helm に転送されます。[`-f`/`--values`](index.md) ファイルを補完します。

| オプション | マーカー | 説明 |
|---|---|---|
| `--set=KEY=VALUE` | `---->` | 値をインラインで上書きします（例: `--set image.tag=1.2.3`）。型は推論されます。繰り返し可能で、後の `--set` が勝ち `-f` ファイルを上書きします。 |
| `--set-string=KEY=VALUE` | `---->` | `--set` と同様ですが、値を常に文字列として保持します（例: `"01"` の保持）。繰り返し可。 |
| `--set-file=KEY=PATH` | `---->` | `PATH` のファイル内容全体を `KEY` の値にします（証明書やスクリプトに便利）。繰り返し可。 |
| `--set-json=KEY=JSON` | `---->` | JSON 式から値を設定し、オブジェクトや配列を表現できます（例: `--set-json 'ports=[80,443]'`）。繰り返し可。 |
| `--set-literal=KEY=VALUE` | `---->` | 書いたとおりに値を設定し、型変換を一切行いません。繰り返し可。 |
| `-f`, `--values=FILE` | `---->` | レンダリングに使われ Helm に転送される YAML 値ファイル。繰り返し可能で順に重ねられます。 |

## Helm グローバルオプション

クラスターへの接続と Helm 環境を設定し、そのまま Helm に転送されます。（`lint` はオフラインのため
クラスター関連オプションはほぼ無関係ですが、一貫性のため受け付けます。）

| オプション | マーカー | 説明 |
|---|---|---|
| `-n`, `--namespace=NAMESPACE` | `---->` | リクエストで想定する名前空間スコープ。既定は現在の kube-context の名前空間です。 |
| `--kube-context=CONTEXT` | `---->` | 使用する kubeconfig 内のコンテキスト。現在のコンテキストを切り替えずに特定のクラスター/ユーザーを対象にできます。 |
| `--kubeconfig=FILE` | `---->` | `$KUBECONFIG` / `~/.kube/config` の代わりに使う kubeconfig ファイルのパス。 |
| `--kube-apiserver=ADDRESS` | `---->` | kubeconfig の API サーバーアドレスとポートを上書きします。 |
| `--kube-as-user=USER` | `---->` | このユーザーを偽装します（RBAC ユーザー偽装）。偽装権限が必要です。 |
| `--kube-as-group=GROUP` | `---->` | このグループを偽装します。複数グループのため繰り返し可。 |
| `--kube-ca-file=FILE` | `---->` | API サーバーの TLS 証明書検証に使う CA 証明書ファイル。 |
| `--kube-token=TOKEN` | `---->` | kubeconfig の資格情報の代わりに認証に使うベアラートークン。 |
| `--kube-tls-server-name=NAME` | `---->` | API サーバー証明書の検証に使うサーバー名（URL のホストと異なる場合）。 |
| `--kube-insecure-skip-tls-verify` | `---->` | API サーバー証明書の検証をスキップします。安全でない——信頼済み/テスト環境のみ。 |
| `--burst-limit=INT` | `---->` | API リクエストのクライアント側スロットリングのバースト上限（既定 `100`）。 |
| `--qps=QPS` | `---->` | API サーバーへのクライアント側の毎秒クエリ上限。小数可。 |
| `--registry-config=FILE` | `---->` | OCI レジストリ設定（資格情報）ファイルのパス。 |
| `--repository-cache=DIR` | `---->` | 依存解決に使うキャッシュ済みリポジトリインデックスのディレクトリ。 |
| `--repository-config=FILE` | `---->` | リポジトリ名と URL を対応づけるファイルのパス（`repositories.yaml`）。 |

## グローバルオプション

すべての[グローバルオプション](index.md)（`--debug`、`--unsafe` など）も利用できます。

## 例

```bash
# カレントディレクトリを lint
kube-kts lint .

# 追加の値ファイルとインライン上書きを伴う厳格 lint
kube-kts lint /path/to/repository ./out --strict -f prod.yaml --set image.tag=1.2.3

# サブチャートを含めて静かに lint
kube-kts lint ./helm --with-subcharts --quiet
```
