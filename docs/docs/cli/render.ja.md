# render

```bash
kube-kts render <REPOSITORY> [TARGET] [-f FILE...]
```

リポジトリをディスク上の標準 Helm Chart にレンダリングします。スキャン・コンパイル・実行の全パイプ
ラインを走らせ、結果を通常の Helm Chart（`Chart.yaml`、`values.yaml`、純粋な YAML から成る
`templates/` ディレクトリ）としてターゲットディレクトリに書き出します。出力に Kotlin や Go テンプレート
ロジックは含まれないため、`helm` で直接使ったり、コミット・差分比較・手動確認ができます。

`render` は Kube KTS の世界と標準ツールとの橋渡しです。Helm ベースのコマンドが内部で行うことは、すべて
このステップから始まります。明示的に実行するのが、Kube KTS の生成物を*見る*最良の方法です。

!!! tip "出力を確認する"
    明示的な `TARGET` を渡すと、コマンド終了後も Chart が保持されます。指定しない場合、Chart は OS が
    後始末する可能性のある一時ディレクトリに書き出されます。

## パラメータ

| パラメータ | 必須 | 説明 |
|---|---|---|
| `REPOSITORY` | はい | レンダリングするリポジトリのパス。省略時はカレントディレクトリを使用します。 |
| `TARGET` | いいえ | Helm Chart の書き出し先ディレクトリ。省略時は一時ディレクトリを作成します。既存のターゲットは新しくレンダリングした Chart で上書きされます。 |

## オプション

[グローバルオプション](index.md)に加えて:

| オプション | マーカー | 説明 |
|---|---|---|
| `-f`, `--values=FILE` | `---->` | レンダリング中に Kotlin スクリプトから利用できる YAML 値ファイル。繰り返し可能で、順に重ねられ後のファイルが前を上書きします。各ファイルは存在する必要があります。（`render` では値は生成される Chart に影響します。`render` は Helm を呼ばないため Helm には渡されません。） |

## 例

```bash
# カレントディレクトリを ./out にレンダリング
kube-kts render . ./out

# 2 つの値ファイルを重ねて特定のリポジトリをレンダリング
kube-kts render /path/to/repository ./out -f base.yaml -f prod.yaml

# 成功するか確認するためだけに一時ディレクトリへレンダリング
kube-kts render ./helm
```
