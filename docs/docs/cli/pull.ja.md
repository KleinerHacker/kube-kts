# pull

```bash
kube-kts pull <CHART> [オプション]
```

`helm pull` を実行してリポジトリからチャートをダウンロードします。リモートのチャートに対して動作するため、**リポジトリもレンダリング手順も不要**です——呼び出しは直接 Helm に転送されます。

## 動作の仕組み

1. リポジトリのスキャン・コンパイル・レンダリングは行われません。KTS スクリプトはここでは無関係です。
2. `helm pull <CHART>` が、転送されるすべてのオプションを付加して実行されます。
3. Helm がチャートをダウンロードします。コマンドの終了コードは Helm の結果を反映します。

## パラメータ

| パラメータ | 必須 | 説明 |
|---|---|---|
| `CHART` | はい | ダウンロードするチャート参照。位置引数 `CHART` として Helm に転送されます。 |

## pull オプション

| オプション | マーカー | 説明 |
|---|---|---|
| `-d`, `--destination=DIR` | `---->` | チャートの書き込み先。`--untardir` と併用時は組み合わされます。 |
| `--prov` | `---->` | provenance ファイルを検証せずに取得します。 |
| `--untar` | `---->` | ダウンロード後にチャートを展開します。 |
| `--untardir=DIR` | `---->` | `--untar` 指定時にチャートを展開するディレクトリ。 |

## チャートダウンロードオプション

| オプション | マーカー | 説明 |
|---|---|---|
| `--repo=URL` | `---->` | チャートを特定するリポジトリ URL。 |
| `--username=USER`、`--password=PASSWORD` | `---->` | リポジトリの認証情報。 |
| `--pass-credentials` | `---->` | すべてのドメインに認証情報を渡します。 |
| `--ca-file=FILE`、`--cert-file=FILE`、`--key-file=FILE` | `---->` | ダウンロードに使用する TLS 資材。 |
| `--insecure-skip-tls-verify` | `---->` | ダウンロードの TLS 証明書チェックをスキップします。 |
| `--plain-http` | `---->` | ダウンロードに安全でない HTTP を使用します。 |
| `--keyring=FILE` | `---->` | `--verify` に使用する公開鍵。 |
| `--verify` | `---->` | 使用前にパッケージを検証します。 |
| `--version=VERSION` | `---->` | 使用するチャートバージョンの制約。 |
| `--devel` | `---->` | 開発版を許可します（`>0.0.0-0` に相当）。 |

## Helm グローバルオプション

すべての [Helm グローバルオプション](status.md#helm-global-options)が Helm に転送されます。

## グローバルオプション

すべての[グローバルオプション](index.md)も利用できます。

## 例

```bash
# 特定のチャートバージョンをダウンロードして展開
kube-kts pull bitnami/nginx --version 15.0.0 --untar -d ./charts
```
