# list

```bash
kube-kts list [オプション]
```

`helm list` を実行してリリースを一覧表示します。`status` と同様に `list` はクラスターに対して動作するため、**リポジトリもレンダリング手順も不要**です——呼び出しは直接 Helm に転送されます。

## 動作の仕組み

1. リポジトリのスキャン・コンパイル・レンダリングは行われません。KTS スクリプトはここでは無関係です。
2. `helm list` が、転送されるすべてのオプションを付加して実行されます。
3. Helm がリリース一覧を表示します。コマンドの終了コードは Helm の結果を反映します。

## list オプション

| オプション | マーカー | 説明 |
|---|---|---|
| `-a`, `--all` | `---->` | フィルターを適用せずすべてのリリースを表示します。 |
| `-A`, `--all-namespaces` | `---->` | すべての名前空間のリリースを一覧表示します。 |
| `-d`, `--date` | `---->` | リリース日でソートします。 |
| `--deployed` | `---->` | デプロイ済みのリリースを表示します。他の状態フィルターがなければ自動的に有効になります。 |
| `--failed` | `---->` | 失敗したリリースを表示します。 |
| `--filter=REGEXP` | `---->` | Perl 互換の正規表現。一致するリリースのみ一覧表示します。 |
| `-m`, `--max=INT` | `---->` | 取得するリリースの最大数。 |
| `--no-headers` | `---->` | デフォルト出力形式でヘッダーを表示しません。 |
| `--offset=INT` | `---->` | 一覧内の次のリリースインデックス。開始位置からのオフセットに使用します。 |
| `-o`, `--output=FORMAT` | `---->` | 出力形式：`table`（既定）、`json`、`yaml`。 |
| `--pending` | `---->` | 保留中のリリースを表示します。 |
| `-r`, `--reverse` | `---->` | ソート順を逆にします。 |
| `-l`, `--selector=SELECTOR` | `---->` | フィルター用のラベルクエリ（`=`、`==`、`!=` をサポート）。 |
| `-q`, `--short` | `---->` | 短い（静かな）一覧形式で出力します。 |
| `--superseded` | `---->` | 置き換えられたリリースを表示します。 |
| `--time-format=FORMAT` | `---->` | Go の時刻フォーマッターで時刻を整形します。 |
| `--uninstalled` | `---->` | アンインストール済みのリリースを表示します（アンインストール時に `--keep-history` を使用した場合）。 |
| `--uninstalling` | `---->` | 現在アンインストール中のリリースを表示します。 |

## Helm グローバルオプション

すべての [Helm グローバルオプション](status.md#helm-global-options)（`-n`/`--namespace`、`--kube-context` など）が Helm に転送されます。

## グローバルオプション

すべての[グローバルオプション](index.md)（`--debug`、`--unsafe` など）も利用できます。

## 例

```bash
# 現在の名前空間のリリースを一覧表示
kube-kts list

# すべての名前空間の全リリースを JSON で一覧表示
kube-kts list -A -o json

# 失敗したリリースのみを新しい順に表示
kube-kts list --failed --date --reverse
```
