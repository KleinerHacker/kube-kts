# validate

```bash
kube-kts validate <REPOSITORY>
```

KTS リポジトリを検証します。リポジトリをスキャンし、すべての `*.spec.kts`、`*.lib.kts`、従来の
YAML/テンプレートファイルを検出し、リポジトリが構造的に正しく、必要なファイル（Chart 定義など）が
そろっているかを確認します。スクリプトのコンパイルや実行は**行わず**、出力成果物も生成しません。純粋に
高速な構造チェックです。

CI パイプラインの最初の、最も安価なゲートとして使えます。より高価なコンパイル/レンダリングの前に、
欠落や配置ミスのファイルを検出し、クラスターや Helm に一切触れません。

!!! tip "validate の位置づけ"
    `validate` は*構造*のみを確認します。Kotlin スクリプトがコンパイル・実行できるかも確認するには
    [`compile`](compile.md) を、Helm YAML も生成するには [`render`](render.md) を使ってください。

## パラメータ

| パラメータ | 必須 | 説明 |
|---|---|---|
| `REPOSITORY` | はい | 検証するリポジトリのパス。省略時はカレントディレクトリを使用します。存在しないパスは即座に失敗します。 |

## オプション

このコマンドは[グローバルオプション](index.md)のみを受け付けます。ここで特に有用なのは `--verbose`
（どのファイルが検出されるか確認）と `--exception`（分かりにくい理由で検証が失敗したときにスタック
トレースを取得）です。

## 例

```bash
# カレントディレクトリを検証
kube-kts validate .

# 特定のリポジトリを検証し、検出された全ファイルを表示
kube-kts validate /path/to/repository --verbose

# CI で検証し、失敗時に完全なスタックトレースを表示
kube-kts validate ./helm --exception
```
