# compile

```bash
kube-kts compile <REPOSITORY> [--unsafe]
```

KTS リポジトリをオブジェクトインスタンスにコンパイルします。[`validate`](validate.md) より一歩進み、
実際に Kotlin スクリプトを実行して、レンダラーが後で YAML に変換する Chart のメモリ上オブジェクト
モデル（chart、deployment、service など）を構築します。出力をまだ書き出さずに、スクリプトが
**コンパイル・実行**できるか（型エラー、null 安全違反、スクリプト内で投げられる実行時例外を含む）を
確認するのに適したコマンドです。

コンパイルはスクリプトを実行するため、スクリプトレベルの問題が最初にここで表面化します。無効な値参照、
失敗した `require(...)`、インポートが許可されていないクラスを参照するスクリプトなどです。

!!! warning "`--unsafe` は任意コードを実行します"
    既定ではスクリプトは制限付きサンドボックスで実行され、`import` 文と完全修飾クラス名が禁止されます。
    必要なら `--unsafe` を追加してください。ただし、スクリプトが任意の JVM コードを実行できるため、
    信頼できるリポジトリでのみ使用してください。

## パラメータ

| パラメータ | 必須 | 説明 |
|---|---|---|
| `REPOSITORY` | はい | コンパイルするリポジトリのパス。省略時はカレントディレクトリを使用します。存在しないパスは即座に失敗します。 |

## オプション

[グローバルオプション](index.md)に加えて、コンパイルはスクリプトを実行する段階なので
[`--unsafe`](index.md) フラグが特に関係します。

| オプション | マーカー | 説明 |
|---|---|---|
| `--unsafe` | `!!!` | スクリプト内で `import` 文と完全修飾クラス名を許可し、追加の JVM クラスを使えるようにします。任意コード実行を有効にするため、信頼できるリポジトリでのみ使用してください。 |

## 例

```bash
# カレントディレクトリをコンパイル
kube-kts compile .

# スクリプトで import を使うリポジトリをコンパイル
kube-kts compile /path/to/repository --unsafe

# スクリプトが例外を投げたら完全なスタックトレースを表示
kube-kts compile ./helm --exception
```
