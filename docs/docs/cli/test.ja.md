# test

```bash
kube-kts test <RELEASE> [オプション]
```

`helm test` を実行してリリースに定義されたテストを実行します。既存のリリースに対して動作するため、**リポジトリもレンダリング手順も不要**です——呼び出しは直接 Helm に転送されます。

## 動作の仕組み

1. リポジトリのスキャン・コンパイル・レンダリングは行われません。KTS スクリプトはここでは無関係です。
2. `helm test <RELEASE>` が、転送されるすべてのオプションを付加して実行されます。
3. Helm がテストフックを実行します。コマンドの終了コードは Helm の結果を反映します。

## パラメータ

| パラメータ | 必須 | 説明 |
|---|---|---|
| `RELEASE` | はい | テストするリリース名。位置引数 `RELEASE` として Helm に転送されます。 |

## test オプション

| オプション | マーカー | 説明 |
|---|---|---|
| `--filter=KEY=VALUE` | `---->` | 属性（例：`name=...`）でテストを選択、または `!attribute=value` で除外します。繰り返し可。 |
| `--hide-notes` | `---->` | テスト出力に notes を表示しません。 |
| `--logs` | `---->` | すべてのテスト完了後にテスト Pod のログを出力します。 |
| `--timeout=DURATION` | `---->` | 個々の Kubernetes 操作を待機する時間。 |

## Helm グローバルオプション

すべての [Helm グローバルオプション](status.md#helm-global-options)が Helm に転送されます。

## グローバルオプション

すべての[グローバルオプション](index.md)も利用できます。

## 例

```bash
# リリースのテストを実行し、ログを表示
kube-kts test my-app --logs

# 特定のテストのみ実行
kube-kts test my-app --filter name=my-test
```
